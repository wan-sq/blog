# 博客编辑须知（给自己看的）

这个博客的文章就是普通 Markdown 文件，不用会编程。用 Obsidian 改完全没问题，守住下面几条就行。

## 文章和图片放哪

- **文章**：`src/content/posts/` 里，一篇一个 `.md` 文件。
- **图片**：先把图丢进 `public/images/`，再在正文里写：
  ```
  ![图片说明](images/文件名.jpg)
  ```

## 文件结构

每篇文章分两段：

1. 最上面两条 `---` 之间是「前置信息」——标题、日期、标签、摘要，一般照抄格式改文字即可：
   ```
   ---
   title: 文章标题
   date: 2026-07-18
   tags: [标签1, 标签2]
   excerpt: 一句话摘要
   ---
   ```
2. 下面就是正文，用普通 Markdown 写。

## 能用的写法（通用 Markdown）

- 标题：`# 一级` `## 二级`
- 加粗 `**字**`、斜体 `*字*`
- 列表：`- 项目` 或 `1. 项目`
- 图片：`![说明](images/x.jpg)`
- 链接：`[文字](https://网址)`

## 千万别踩的坑

- ❌ **别用单个方括号 `[文字]` 包普通内容**。Obsidian 会把它当成「内部链接」标红，看着像报错（其实不是）。要写提示就用**圆括号 `（...）`**。
- ❌ **别用 Obsidian 的专属语法**，博客渲染器不认，会原样显示成乱码：
  - `[[双括号]]` 跳转笔记
  - `![[嵌入]]` 嵌入笔记
  - `> [!note]` 这类 callout 折叠块
- ⚠️ 注意：文章顶部 `tags: [标签]` 里的方括号、图片语法 `![](...)` 里的方括号是**正常的**，不用动。

## 改完怎么发布

1. 打开 **GitHub Desktop**，选中 `blog-wan-sq` 仓库。
2. 左下角写一句说明（比如「更新胖东来日记」）→ 点 **Commit to main**。
3. 再点右上角 **Push**。
4. 等几分钟，GitHub Actions 自动部署好，线上就更新了：
   https://wan-sq.github.io/blog

## 想本地先看效果（可选）

在仓库目录打开终端，跑：

```
npm run dev
```

浏览器打开它给出的本地地址（一般是 http://localhost:5173）就能预览，改文件会实时刷新。看完 `Ctrl + C` 关掉。
