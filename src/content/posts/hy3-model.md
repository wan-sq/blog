我是搞科研的，日常离不开 AI——写代码、查资料、整点花活。最近被安利了 hy3，用了一阵，记录点真实的体感。

先澄清一个容易误会的点：**hy3 本身也是大模型**，不是什么"机器人"或"独立程序"。它和你在别处聊的对话模型是同一类东西——区别只在于，它跑在 WorkBuddy 里，外面被套了一层能调工具、能自己跑流程的 agent 外壳。所以它能答，也能做。这篇不跟谁比，就单聊聊它。

## 一、hy3 到底是什么

拆开看就两层：

- **底层是混元大模型**：负责"想"——理解你的话、规划怎么做、生成文字和代码。
- **外层是 WorkBuddy 的 agent 框架**：负责"动手"——给它接上了命令行、文件系统、浏览器，还有一堆可插拔的 MCP 工具（比如登录态管理、文献库、各种 API）。

所以当你说"帮我搭个博客发到 GitHub Pages"，它干的不是"给你一段代码让你自己跑"，而是真的去建 Vite + React 项目、写路由、配 GitHub Actions、把站点部署上去，中间自己跑命令、读报错、改文件。这一层"执行"，是裸对话模型在对话框里接不住的——你得自己当那个把每一步串起来的胶水，而 hy3 这层外壳补上了这块。

> 一句话：**hy3 = 混元大模型 + WorkBuddy 的智能体工具外壳**。聪明的是模型，能干活的是那层外壳。

## 二、我这阵子拿它真干成的事

光说概念没意思，列几个我亲历的：

- **从零搭了个个人博客**：Vite + React + React Router，文章用 markdown 写、丢个 .md 就自动上线，GitHub Pages 自动部署。全程我没碰过一行构建配置。
- **抠官方高清图**：我想给文章配《沧元图》的官博原图，它直接把微博登录态接上，从 CDN 把 1920×1080 的原图拽下来，替换掉之前那些糊的聚合图。
- **自动发知乎**：写好文章，它走浏览器把正文导进知乎专栏、校验排版（标题/引用/列表都对），我最后点一下发布就行。
- **把流程固化成技能**：因为发博客、发知乎这俩事会反复干，它顺手建了两个可复用 skill——下次新开一个任务，不用从头教，直接调就行。

这几件事的共同点是：**跨多个工具、跨多步、还要真去操作我的文件系统和网络**。这种"连贯动手"的活，纯对话模型在对话框里是接不住的。

## 三、用下来的真实体感

说几个它让我觉得"有点东西"的地方：

- **长上下文是真的长**：从搭项目到配部署到发文章，一整条链路它都记着，不会聊到一半把前面的约定忘了。
- **会自己规划多步任务**：你给个目标，它会拆成"先干嘛后干嘛"，卡住了自己读报错、换条路重试，不是只会等你下指令。
- **有记忆、会沉淀**：它干过的活能写成 skill 存下来。第一次慢，第二次就快了——这点纯对话模型给不了，你每次都得从头把背景交代一遍。
- **能调用外部工具**：浏览器自动化、命令行、API、登录态，这些让它不只是"聊天框"，而是个能在我电脑上真干活的东西。

## 四、说点不足，不藏着

用下来也有劝退的地方，坦白讲：

1. **慢且偶尔绕弯**。它能干活，但有时候为了插张图，在知乎编辑器里反复试、卡半天——这时候你真想自己上手。后来我学乖了：这种"细活"我自己点一下更快。
2. **环境依赖多**。它要跑命令、要代理、要登录态，出问题的地方比纯网页聊天多。新手第一次配会有点懵。
3. **偶尔"跑飞"**：一旦开始自己跑，极少情况下会绕远路、卡在某个工具索引上（别问我怎么知道的，知乎那个 Draft.js 编辑器差点把我整自闭）。这时候需要人兜个底。

但反过来说，它干过的活会沉淀成技能，第一次慢、第二次就快。这个复利效应，是它最值钱的地方。

## 五、到底怎么用好它

我的经验是：**把"把这件事做成"交给它，把"想清楚/写好"留给自己把关**。

- 重活——搭站、部署、自动化发布、串起一堆工具——放心丢给它跑，你搭把手兜底就行。
- 细活和最后拍板——比如点发布、确认排版——你自己来，比等它磨蹭更省心。
- 反复干的事——让它固化成 skill，别每次都重新教。
