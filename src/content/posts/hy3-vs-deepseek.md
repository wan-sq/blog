我是搞科研的，日常离不开 AI——写代码、查资料、整点花活。最近被安利了 hy3（其实就是 WorkBuddy 里跑的这个大模型），断断续续用了一阵，顺手把它和我天天在用的 DeepSeek 拉到一起比了比。

先说结论：**它俩底层都是大模型，别被名字骗了。** 真正的差别不在"聪不聪明"，而在"被怎么用"——DeepSeek 你直接跟裸模型聊、自己动手落地；hy3 是同一个量级的大模型，被 WorkBuddy 套了一层能调工具、能自己跑流程的 agent 外壳。所以它能答，也能做。下面我用自己这一周干的事，给你掰开说。

## 一、最直观的区别：一个裸用，一个带"工具手臂"

DeepSeek 我用了很久，体验是：你问它，它答；你让它写段代码，它吐一段给你，复制粘贴是你的事。它的长处在"在对话框里把事想清楚"。

hy3 底层也是大模型，硬实力并不比 DeepSeek 差；差别在于它跑在 WorkBuddy 里——这个外壳给它接上了命令行、文件系统、浏览器、还有各种 MCP 工具。所以我说"给我搭个博客发到 GitHub Pages 上"，它不光给方案，还真去建了 Vite + React 项目、写了路由、配了 GitHub Actions、把域名部署上去了，中间自己跑命令、读报错、改文件，我基本就在旁边看着。

这不是"它更会写代码"——而是它的工作闭环里本来就包含"执行"这一步。DeepSeek 给你方案，hy3 连方案带落地一起交。

> 一句话：DeepSeek 是裸模型，hy3 是"大模型 + 工具手臂"。

## 二、我这周拿 hy3 真干成的事

光说概念没意思，列几个我亲历的：

- **从零搭了个个人博客**：Vite + React + React Router，文章用 markdown 写、丢个 .md 就自动上线，GitHub Pages 自动部署。全程我没碰过一行构建配置。
- **抠官方高清图**：我想给文章配《沧元图》的官博原图，它直接把微博登录态接上，从 `sinaimg.cn` 的 CDN 把 1920×1080 的原图拽下来，替换掉之前那些糊的聚合图。
- **自动发知乎**：写好文章，它走浏览器把正文导进知乎专栏、校验排版（标题/引用/列表都对），我最后点一下发布就行。
- **把流程固化成技能**：因为发博客、发知乎这俩事会反复干，它顺手建了两个可复用 skill——下次新开一个任务，不用从头教，直接调就行。

这几件事的共同点是：**跨多个工具、跨多步、还要真去操作我的文件系统和网络**。这种"连贯动手"的活，纯对话的裸模型（不管它多聪明）在对话框里是接不住的——你得自己当那个把每一步串起来的胶水。而 hy3 这层 agent 外壳，正好补上了"动手"这块。

## 三、公平起见，DeepSeek 赢的地方也得说

我也不吹 hy3。有几个场景我宁愿开 DeepSeek：

- **纯粹的写作/问答**：长文润色、知识讲解、头脑风暴，DeepSeek 的语感更顺、更不容易"车轱辘话"，而且响应快、不挑环境。
- **轻量、即开即用**：开个网页就能聊，不用等它去"建项目、跑命令"。很多小问题，DeepSeek 三秒给答案，hy3 可能还在琢磨怎么动手（毕竟它默认想帮你把事做完，动静大一点）。
- **稳定可控**：hy3 一旦开始"自己跑"，偶尔会绕远路、卡在某个编辑器或者工具索引上（别问我怎么知道的，知乎那个 Draft.js 编辑器差点把我整自闭）。DeepSeek 就不会有这种"它跑飞了"的意外——因为它压根不会自己动你的环境。

## 四、一张表看明白

| 维度 | hy3（WorkBuddy 里的大模型） | DeepSeek（裸用大模型） |
| --- | --- | --- |
| 核心定位 | 大模型 + WorkBuddy 智能体框架（可调用工具） | 大模型（纯对话，裸用） |
| 动手干活 | 建项目/跑命令/部署/发帖，自己干 | 给方案，落地靠你 |
| 写作语感 | 够用，但偏"执行导向" | 更顺、更稳 |
| 上手速度 | 首次要配环境，慢热 | 开页即聊 |
| 多工具串联 | 强项 | 弱（得你当胶水） |
| 可控性 | 偶尔跑飞，需要兜底 | 稳（不碰你的环境） |
| 适合 | "帮我把这个东西做出来" | "帮我想想/写写这个" |

## 五、说点大实话：hy3 的痛点

我不藏着。用下来最劝退的有两点：

1. **慢且偶尔绕弯**。它能干活，但有时候为了插张图，在知乎编辑器里反复试、卡半天——这时候你真想自己上手。后来我学乖了：这种"细活"我自己点一下发布更快。
2. **环境依赖**。它要跑命令、要代理、要登录态，出问题的地方比纯网页聊天多。新手第一次配会有点懵。

但好处是：**它干过的活会沉淀成技能**。第一次慢，第二次就快了。这点纯对话模型给不了——你每次都得从头把背景交代一遍（虽然 DeepSeek 也能记住上下文，但跨任务的"肌肉记忆"还是得靠 agent 侧固化）。

## 六、到底谁适合用哪个

- 如果你想要的是**"帮我把这件事做成"**——搭站、部署、自动化发布、串起一堆工具——闭眼选 hy3 这种"带工具的大模型"。
- 如果你想要的是**"帮我想清楚 / 写好 / 答上来"**——写作、答疑、脑暴——DeepSeek 更舒服、更省心，而且它本身也是大模型，真要写东西一点不虚。
- 我自己的常态是：**hy3 当执行层，DeepSeek 当思考层**。重活交给它跑，纯聊交给它聊，配合着来最爽。
